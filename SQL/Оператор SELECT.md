Оператор **SELECT (ВЫБРАТЬ)** языка SQL является самым важным и наиболее часто используемым оператором. Он предназначен для ***выборки информации из таблиц базы данных***. Синтаксис оператора **SELECT**:
```sql
SELECT [DISTINCT] <список выражений надо атрибутами и константами>
	FROM <список таблиц>
	[WHERE <условие выборки>]
	[GROUP BY <список атрибутов>]
	[HAVING <условие>]
	[UNION <выражение с оператором SELECT>]
	[ORDER BY <список атрибутов>];
```
Ключевое слово **SELECT** сообщает базе данных, что данное предложение является запросом на *выборку информации*. После слова **SELECT** через запятую перечисляются *наименования полей* (список атрибутов), содержимое которых запрашивается.

Обязательным ключевым словом в предложении-запросе **SELECT** является слово **FROM (ИЗ)**. За ключевым словом **FROM** указывается список таблиц разделенных запятыми имен таблиц, из которых извлекается информация. Например,
```sql
SELECT NAME, SURNAME
	FROM STUDENT;
```
> ***Любой SQL-запрос должен заканчиваться символом ";" (точка с запятой)***

Если необходимо вывести значения всех столбцов таблицы, то можно вместо перечисления их имен использовать символ "*" (звездочка).
```sql
SELECT *
	FROM STUDENT;
```

> **Замечание**<br>
> Получаемые в результате SQL-запроса таблицы не в полной мере отвечают определению реляционного отношения. В частности, в них могут оказаться кортежи с одинаковыми значениями атрибутов.

Для исключения из результата **SELECT**-запроса повторяющихся записей используется ключевое слово **DISTINCT (ОТЛИЧНЫЙ)**, т.е. **DISTINCT** *исключает* дубликаты строк, в которых значения *всех* выбранных полей идентичны.

Ключевое слово **ALL (ВСЕ)**, в отличии от **DISTINCT**, оказывается противоположное действие, т.е. при его использовании повторяющиеся строки *включаются* в состав выходных данных.
> Режим **ALL** действует по умолчанию, поэтому в запросах оно практически не используется

Использование в операторе **SELECT** предложения, определяемого ключевым словом **WHERE (ГДЕ)**, позволяет задавать выражение условия (*предикат*), принимающее значение ***истина*** или ***ложь*** (а также ***неизвестно*** при использовании [[Типы данных#Неопределенные или отсутствующие данные (NULL)|NULL]]) для значений полей строк таблиц, к которым обращается оператор **SELECT**.<br>
Предложение **WHERE** определяет, <i><b>какие строки</b> указанных таблиц должны быть выбраны.</i> В таблицу-результат включаются только те строки, для которых условия (*предикат*), указанное в предложении **WHERE**, принимает значение ***истина***.

### Порядок выполнения операторов в SELECT
![[Порядок выполнения операторов в SELECT.png]]<br>
*Рисунок 1* - Порядок выполнения операторов в SELECT

1. **FROM** - определяет источники (таблицы или подзапросы)
2. **WHERE** - фильтрует строки на основе заданных условий
3. **GROUP BY** - группирует строки по значениям одного или нескольких столбцов
4. **HAVING** - фильтрует группы, созданные с помощью **GROUP BY**
5. **SELECT** - выбирает столбцы и выражения для вывода, включая вычисляемые поля
6. **ORDER BY** - сортирует результирующий набор по указанным столбцам
7. **Оконный функции** - выполняет вычисления для выделенного набора строк
