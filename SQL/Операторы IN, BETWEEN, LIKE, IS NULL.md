При задании логического условия в предложении <b>WHERE</b> могут быть использованы операторы:
- <b>IN</b> - предикат (условие) с <b>IN</b> считается истинным, если значение поля, имя которого указано слева от <b>IN</b>, в точности *совпадает* с одним из значений, перечисленных в списке, указанном в скобках справа от **IN**. Также может быть построен предикат **NOT IN**, который считается истинным, если значение поля слева *не совпадает* ни с одним из значений, принадлежащим списку, указанному в скобках справа от **NOT IN**. Пример:
```sql
SELECT *
FROM EXAM_MARKS
WHERE MARK IN (4, 5);
```
- <b>BETWEEN</b> - используется для проверки условия вхождения значения поля в заданный интервал, т.е. вместо списка значений этот оператор задает границы его изменения, причем границы *входят* во множество значений. Пример:
- ```sql
  SELECT *
  FROM SUBJECT
  WHERE HOUR BETWEEN 30 AND 40;
  ```
- <b>LIKE</b> - применим только для символьных типов **CHAR** или **VARCHAR** и другие. Этот оператор осуществляет просморт строковых значений полей с целью определения, входит ли заданная в операторе **LIKE** подстрока (образец поиска) в символьную строку, являющуюся значением проверяемого поля. Для осуществления выборки используются шаблоны искомого образца строки, использующие символы:
	- символ подчеркивания "\_" - допускает наличия в указанном месте *одного любого символа*
	- символ "%" - допускает наличие в указанном месте
```sql
/*
Выбор студентов, у которых фамилии начинаются на букву "P"
*/
SELECT *
FROM STUDENT
WHERE SURNAME LIKE 'P%';
```
- Для отключения служебных функций символов "\_" и "%" перед ними нужно поставить escape-символ (escape character) . В SQL в качестве такого символа может быть назначен любой символ. Для этих целей предназначено ключевое слово **ESCAPE**.
```sql
/*
Первый "_" является служебным (т.е. будет соответствовать любому символу в проверяемой строке), второй "_" интерпретируется как обычный символ, так же как "P"
*/
SELECT *
FROM STUDENT
WHERE SURNAME LIKE '_\_P' ESCAPE '\';
```
- <b>IS NULL</b> - используется для проверки содержимого поля на наличие (отсутствие) в нем пустого значения **NULL**, т.е. **IS NULL** (ЯВЛЯЕТСЯ ПУСТЫМ) или **IS NOT NULL** (НЕ ЯВЛЯЕТСЯ ПУСТЫМ).
